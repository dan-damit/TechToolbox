## Overview

`Get-FilesUsingKeywords` recursively searches files under a given directory for lines matching any of the supplied keywords. The search always runs asynchronously: a PowerShell runspace is used for local execution and `Invoke-Command -AsJob` is used for remote execution. Either way, `Wait-TerminalState` drives the poll/spinner loop until completion.

---

## Parameters

| Parameter | Type | Description |
|----------|------|-------------|
| **Path** | `string` | The directory to search. Required. |
| **Keywords** | `string[]` | One or more keywords (or regex patterns) to search for within file contents. Required. |
| **ComputerName** | `string` | Remote computer to run the search on. When omitted the search runs locally. |
| **Credential** | `pscredential` | Credentials used when connecting to the remote machine or accessing a protected directory path. |
| **FileFilter** | `string` | Wildcard filter applied to file names (e.g., '*.log', '*.txt'). Defaults to '*' (all files). |
| **NoRecurse** | `switch` | When specified, only the top-level directory is searched instead of recursing into subdirectories. |
| **CaseSensitive** | `switch` | Treat keyword matching as case-sensitive. |
| **SimpleMatch** | `switch` | Treat keywords as literal strings rather than regular expressions. |
| **UseSsh** | `switch` | Use SSH transport instead of WSMan when connecting to remote hosts. |
| **UseCredSSP** | `switch` | Use CredSSP for WSMan remoting. |
| **SshPort** | `int` | SSH port when -UseSsh is specified. Defaults to 22. |
| **TimeoutSeconds** | `int` | Maximum time in seconds to wait for the search to complete. Range: 10-86400. Defaults to 1200. |
| **PollSeconds** | `int` | Interval in seconds between progress polls. Range: 1-3600. Defaults to 30. |

## How It Works

### Local Search
On the local machine, it uses a PowerShell runspace to asynchronously scan files matching the filter for the specified keywords.

### Remote Search
For remote computers:
- Uses `Invoke-Command -AsJob` to execute the search on the target host.
- Supports WSMan (with optional CredSSP) or SSH transport.
- Waits for job completion using a configurable timeout and poll interval.

---

## Examples

### **Search local directory for keywords**
```powershell
Get-FilesUsingKeywords -Path 'C:\Logs' -Keywords 'error', 'warning'
```

### **Search with file filter**
```powershell
Get-FilesUsingKeywords -Path 'C:\Data' -Keywords 'password' -FileFilter '*.txt'
```

### **Remote search using SSH**
```powershell
Get-FilesUsingKeywords -ComputerName 'RemotePC01' -Path 'D:\Logs' -Keywords 'fail' -UseSsh -UserName 'admin' -KeyFilePath 'C:\keys\id_rsa'
```

### **Non-recursive search with case sensitivity**
```powershell
Get-FilesUsingKeywords -Path 'C:\Config' -Keywords 'setting' -NoRecurse -CaseSensitive
```

---

## Notes

- Keywords can be plain strings or regular expressions. Use `-SimpleMatch` to force literal string matching.
- Remote searches require appropriate permissions and configured remoting (WSMan/SSH) on the target host.
- The function supports pipeline input for ComputerName.
- Large directories may take significant time; adjust TimeoutSeconds as needed.

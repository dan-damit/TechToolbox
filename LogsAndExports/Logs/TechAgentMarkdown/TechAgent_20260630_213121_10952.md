# Set-OneTimeReboot

## SYNOPSIS
Schedules or removes a one-time reboot task locally or on remote computers.

## DESCRIPTION
The `Set-OneTimeReboot` function creates, updates, or removes a Windows Scheduled Task that runs `shutdown.exe /r` one time at a target local date/time. 

### Key Features:
- **Local Scheduling**: When `-ComputerName` is not provided, the task is scheduled on the local machine.
- **Remote Scheduling**: Supports remote scheduling over PowerShell remoting (WSMan or SSH) when `-ComputerName` is specified.
- **Time Resolution**:
  - If `-DateTime` is supplied, that exact local timestamp is used.
  - If `-DateTime` is omitted, the `-At` parameter (HH:mm format) is used for the next occurrence. If today's time has passed, scheduling rolls to tomorrow.
- **Idempotent Task Update**: If a task with the same `-TaskName` already exists, it is removed and recreated with the new settings.
- **Remove Mode**: Pass `-Remove` to unregister the task instead of creating it.
- **Configuration Defaults**: Unbound parameters can be populated from `settings.oneTimeReboot` in the TechToolbox config, including `At`, `SelfDelete`, `defaultComment`, `taskNamePrefix`, and optional `deleteExpiredTaskAfterMinutes`.
- **Safe Execution**: Supports `-WhatIf` and `-Confirm` for safe preview of operations.

## PARAMETERS

### -ComputerName
Target computer name(s) for remote scheduling/removal. Accepts values from the pipeline and property name input.
- **Type**: `string[]`
- **Aliases**: CN, Host, Name
- **Default Value**: Local machine if omitted.

### -DateTime
Exact local date/time to use for scheduling.
- **Type**: `Nullable[datetime]`
- **Description**: If provided, this exact timestamp is used. Declared before Credential to allow positional binding of the second argument as DateTime in calls like `Set-OneTimeReboot -ComputerName PC01 '4/21/2026 1:00:00 AM'`.

### -Credential
Credential for remote session creation (WSMan/SSH). Also supports username-only for SSH key auth via PSRemoting helper.
- **Type**: `pscredential`

### -At
Time in HH:mm format to schedule the reboot if `-DateTime` is not provided. Uses the next occurrence of this time.
- **Type**: `string`
- **Validation Pattern**: `^(?:[01]\d|2[0-5]):[0-5\d]$`

### -TaskName
Name of the scheduled task. If omitted, a default name is generated using `-taskNamePrefix` from config or a standard prefix.
- **Type**: `string`

### -Comment
Description for the scheduled task.
- **Type**: `string`

### -SelfDelete
If specified, the scheduled task will be deleted after it runs successfully.
- **Type**: `SwitchParameter`

### -WakeToRun
If specified, the computer will wake from sleep to run the reboot task. Requires hardware support.
- **Type**: `SwitchParameter`

### -ForceCloseApps
If specified, applications will be forced closed if they are running at the time of the reboot.
- **Type**: `SwitchParameter`

### -Remove
If specified, unregisters the existing scheduled task instead of creating or updating it.
- **Type**: `SwitchParameter`

### -WhatIf
Shows what would happen if the cmdlet runs. The cmdlet is not run.
- **Type**: `SwitchParameter`

### -Confirm
Prompts you for confirmation before running the cmdlet.
- **Type**: `SwitchParameter`

## INPUTS
`string[]` (ComputerName)

## OUTPUTS
None or Microsoft.PowerShell.Commands.ScheduledTaskInfo depending on operation.

## EXAMPLES

### Example 1: Schedule a local reboot at a specific time
```powershell
Set-OneTimeReboot -DateTime '2026-07-01 02:30:00'
```
This schedules the current computer to reboot on July 1, 2026, at 2:30 AM.

### Example 2: Schedule a remote reboot using time pattern
```powershell
Set-OneTimeReboot -ComputerName PC01 -At '03:00' -Credential (Get-Credential)
```
This schedules PC01 to reboot at the next occurrence of 3:00 AM, prompting for credentials.

### Example 3: Remove a scheduled reboot task
```powershell
Set-OneTimeReboot -ComputerName PC01 -TaskName 'MyReboot' -Remove
```
This removes the scheduled task named 'MyReboot' from PC01.

## NOTES
- Requires appropriate permissions to create/modify scheduled tasks on target machines.
- For remote scheduling, PowerShell Remoting must be enabled and configured correctly.
- The function honors `-WhatIf` and `-Confirm` scopes for both local and remote operations.

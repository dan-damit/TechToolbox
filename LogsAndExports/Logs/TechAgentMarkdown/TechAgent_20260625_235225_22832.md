## Remove-Printers

### SYNOPSIS
Removes all printers from the system, with optional removal of ports, drivers, and per-user mappings.

### SYNTAX
```
Remove-Printers [-IncludePorts] [-IncludeDrivers] [-Force] [-AllUsers] [-PassThru] [-WhatIf] [-Confirm]
[<CommonParameters>]
```

### DESCRIPTION
The `Remove-Printers` function removes all printer queues from the local system. It utilizes Win32_Printer (CIM) to remove queues after resetting the print spooler service and clearing the spool folder.

This cmdlet supports optional removal of TCP/IP printer ports and printer drivers. It includes fallback mechanisms for provider issues and frees common process locks such as `splwow64` and `PrintIsolationHost`. Additionally, it can remove per-user network printer connections across all user profiles on the system.

### PARAMETERS

#### -IncludePorts
Also removes TCP/IP printer ports (non-standard). Use with caution as this may affect other services relying on these ports.

- **Type:** SwitchParameter
- **Required?** False
- **Position?** Named
- **Default value:** None

#### -IncludeDrivers
Removes printer drivers after the queues are gone. This helps in cleaning up unused driver packages from the system.

- **Type:** SwitchParameter
- **Required?** False
- **Position?** Named
- **Default value:** None

#### -Force
Performs a best-effort forced cleanup of driver packages via `pnputil` if standard removal fails. This is useful for stubborn drivers that do not remove cleanly.

- **Type:** SwitchParameter
- **Required?** False
- **Position?** Named
- **Default value:** None

#### -AllUsers
Attempts to remove per-user network printer connections for all user profiles on the system, not just the current user.

- **Type:** SwitchParameter
- **Required?** False
- **Position?** Named
- **Default value:** None

#### -PassThru
Outputs a summary object containing counts of removed printers, ports, drivers, and any failures encountered during the process.

- **Type:** SwitchParameter
- **Required?** False
- **Position?** Named
- **Default value:** None

#### -WhatIf
Shows what would happen if the cmdlet runs. The cmdlet is not run.

#### -Confirm
Prompts you for confirmation before running the cmdlet.

### INPUTS
None. You cannot pipe objects to Remove-Printers.

### OUTPUTS
System.Management.Automation.PSCustomObject (if -PassThru is specified). A summary object with counts and failures.

### EXAMPLES

#### Example 1: Full Cleanup
```powershell
Remove-Printers -IncludePorts -IncludeDrivers -Force -AllUsers -PassThru
```
This command removes all printers, TCP/IP ports, and drivers. It forces cleanup of stubborn drivers via `pnputil`, removes per-user connections for all users, and outputs a summary object.

#### Example 2: Preview Mode
```powershell
Remove-Printers -WhatIf
```
This command shows what would happen if the cmdlet ran without actually making any changes.

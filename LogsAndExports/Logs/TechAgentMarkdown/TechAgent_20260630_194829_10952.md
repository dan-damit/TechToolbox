# Get-LocalAdminMembers

## Synopsis
Gets local Administrators members with SID + metadata, optionally expanding nested groups.

## Description
The `Get-LocalAdminMembers` function retrieves the members of a specified local group on one or more computers. By default, it targets the 'Administrators' group on the local computer. It returns detailed information including Security Identifiers (SIDs) and object metadata. The function supports recursive expansion of nested groups to provide a comprehensive view of effective membership.

## Syntax
```powershell
Get-LocalAdminMembers [-ComputerName <String[]>] [-Group <String>] [-Recurse] [-MaxDepth <Int32>] [-IncludeGroups [<Boolean>]] [-Credential <PSCredential>] [<CommonParameters>]
```

## Parameters

### -ComputerName
One or more computers to query. Accepts pipeline input by property name.
- **Type:** String[]
- **Default Value:** $env:COMPUTERNAME (the local computer)
- **Required:** False
- **Pipeline Input:** True (ByPropertyName, ByValue)

### -Group
The name of the local group to query for members. Defaults to 'Administrators'.
- **Type:** String
- **Default Value:** Administrators
- **Required:** False
- **Pipeline Input:** False

### -Recurse
Expand nested groups (both domain and local) best-effort. When specified, the function will attempt to resolve members of any group found within the target group.
- **Type:** Switch
- **Default Value:** Off
- **Required:** False
- **Pipeline Input:** False

### -MaxDepth
Maximum recursion depth for nested groups when `-Recurse` is used. Prevents infinite loops in deeply nested or circular group structures.
- **Type:** Int32
- **Range:** 0 to 50
- **Default Value:** 5
- **Required:** False
- **Pipeline Input:** False

### -IncludeGroups
Include group objects themselves in the output, not just their members. When true (default), groups found during recursion are listed as entries.
- **Type:** Boolean
- **Default Value:** True
- **Required:** False
- **Pipeline Input:** False

### -Credential
Optional credential for remoting to remote computers. If omitted, the current user's credentials are used.
- **Type:** PSCredential
- **Default Value:** Current User
- **Required:** False
- **Pipeline Input:** False

## Inputs
System.String (ComputerName)

You can pipe a computer name string to `Get-LocalAdminMembers`.

## Outputs
PSCustomObject
The function returns custom objects containing member details such as Name, SID, ObjectClass (User/Group), and potentially nested membership information if `-Recurse` is used.

## Examples

### Example 1: Get local admin members
```powershell
Get-LocalAdminMembers
```
Gets the members of the local Administrators group on the current computer.

### Example 2: Get remote admin members
```powershell
Get-LocalAdminMembers -ComputerName "Server01", "Workstation05"
```
Gets the members of the Administrators group on Server01 and Workstation05.

### Example 3: Recursively expand nested groups
```powershell
Get-LocalAdminMembers -Recurse -MaxDepth 3
```
Gets local admin members and expands any nested groups up to a depth of 3 levels.

## Notes
- The function relies on WMI/CIM or ADSI methods depending on the target system's availability.
- When querying remote computers, ensure that PowerShell remoting is enabled or appropriate firewall rules are in place.
- Use `-Credential` if you need to authenticate as a different user for remote queries.

# Get-InstalledPrinters

Gets printers installed for the currently logged-on user on a remote computer.

---

## Overview

`Get-InstalledPrinters` queries one or more remote computers to identify printers visible to the active interactive user. It uses `Start-NewPSRemoteSession` to establish a connection, detects the current user session, and correlates registry-based printer connections (from HKEY_USERS) with Win32_Printer data. This provides both machine-level and per-user printer mappings.

---

## How It Works

### 1. Remote Session Creation
Establishes a PowerShell session to the target computer(s) using either WSMan or SSH, depending on the `-UseSsh` flag and provided credentials.

### 2. User Detection
Identifies the active interactive user on the remote host by querying the registry for logged-in sessions.

### 3. Printer Resolution
Retrieves printer connections from the detected user's HKEY_USERS hive (registry) and correlates them with Win32_Printer WMI output to provide a comprehensive list of installed printers.

---

## Examples

### **Get printers for the local machine**
```powershell
Get-InstalledPrinters -ComputerName $env:COMPUTERNAME
```

### **Get printers for a remote computer using SSH**
```powershell
Get-InstalledPrinters -ComputerName 'RemotePC01' -UseSsh -UserName 'admin' -KeyFilePath 'C:\keys\id_rsa'
```

### **Get printers for multiple computers with custom credentials**
```powershell
$cred = Get-Credential
Get-InstalledPrinters -ComputerName 'PC01', 'PC02' -Credential $cred
```

---

## Notes

- Requires PowerShell remoting (WSMan) or SSH configured on the target computers.
- The function relies on detecting the active interactive user; if no user is logged in, results may be limited to machine-level printers.
- Supports `-WhatIf` and `-Confirm` via `[CmdletBinding]`.

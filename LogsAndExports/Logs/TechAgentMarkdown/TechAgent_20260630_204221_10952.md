## Get-SystemSnapshot

### Synopsis
Retrieves a system snapshot from one or more remote computers.

### Description
The `Get-SystemSnapshot` function acts as a thin controller that resolves configuration, prepares a helper package, and executes the worker locally or remotely via `Invoke-TTRemoteWorker`. It supports dot-sourcing the worker and calling its entry point. Optionally, it can export the collected data to CSV files.

### Syntax
```powershell
Get-SystemSnapshot [-ComputerName] <String[]> [[-Report]] [[-Credential] <PSCredential>] [[-IncludeServices]] [[-IncludeRoles]] [[-OutDir] <String>] [[-NoExport]] [[-PreferPS7]] [<CommonParameters>]
```

*   **Default value:** None

### Examples

#### Example 1: Basic Snapshot
```powershell
Get-SystemSnapshot -ComputerName "Server01"
```
This command retrieves a system snapshot from Server01 using default settings.

#### Example 2: Snapshot with Credentials and Report
```powershell
$cred = Get-Credential
Get-SystemSnapshot -ComputerName "Server01", "Server02" -Credential $cred -Report
```
This command retrieves snapshots from two servers using provided credentials and displays the interactive report upon completion.

#### Example 3: Snapshot with Services and Roles Included
```powershell
Get-SystemSnapshot -ComputerName "DC01" -IncludeServices -IncludeRoles -OutDir "C:\Snapshots"
```
This command retrieves a snapshot from DC01, including service and role information, and exports the data to C:\Snapshots.

#### Example 4: Snapshot without Export
```powershell
Get-SystemSnapshot -ComputerName "WebServer01" -NoExport
```
This command retrieves a snapshot from WebServer01 but skips CSV export.

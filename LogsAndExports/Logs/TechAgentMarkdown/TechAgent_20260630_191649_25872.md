# Get-InstalledPrinters

Gets printers installed for the currently logged-on user on a remote computer.

---

## Overview

`Get-InstalledPrinters` queries one or more remote computers to identify printers visible to the active interactive user. It leverages `Start-NewPSRemoteSession` to establish connectivity, detects the current user session, and correlates registry-based printer connections (from HKEY_USERS) with Win32_Printer WMI objects. This ensures both machine-level and per-user printer mappings are returned.

---

## Parameters

| Parameter | Type | Description |

| **KeyFilePath** | `string` | Path to the SSH private key file when using key-based authentication. |

## How It Works

### 1. Remote Session Creation
Establishes a PowerShell session to the target computer(s) using either WSMan or SSH, depending on the `-UseSsh` flag and provided credentials.

### 2. User Detection
Identifies the active interactive user session on the remote host to determine which printer context to query.

### 3. Printer Resolution
- Queries `Win32_Printer` for system-wide printers.
- Reads the user's HKEY_USERS registry hive for per-user printer connections.
- Correlates these sources to provide a comprehensive list of visible printers.

---

## Examples

### **Get printers on local machine**
```powershell
Get-InstalledPrinters -ComputerName $env:COMPUTERNAME
```

### **Get printers on remote computer using SSH**
```powershell
Get-InstalledPrinters -ComputerName 'RemotePC01' -UseSsh -UserName 'admin' -KeyFilePath 'C:\keys\id_rsa'
```

### **Get printers on multiple computers with custom credentials**
```powershell
$cred = Get-Credential
Get-InstalledPrinters -ComputerName 'PC01', 'PC02' -Credential $cred
```

---

## Notes

- Requires appropriate permissions to read event logs and registry keys on remote targets.
- Remote execution requires PowerShell remoting enabled (WSMan) or SSH configured correctly.
- The function relies on `Start-NewPSRemoteSession` for connection management.

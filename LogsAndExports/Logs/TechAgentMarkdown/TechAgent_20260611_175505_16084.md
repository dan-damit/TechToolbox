### What Was Changed

The root cause was direct indexed access to `$step.PSObject.Properties['X']` which throws when:
- `$step` is a hashtable (no PSObject support)
- `$step` is null
- The property accessor itself fails in edge cases

The fix introduces two nested helper functions:

**1. Safe-HasProperty** - Safely checks if an object has a given property name:
- Uses `$Obj.PSObject.Properties.Name` (array of names) with `-contains` instead of direct indexed access
- Wraps the operation in try/catch to handle any unexpected failures
- Returns `$false` for null objects or on error

**2. Safe-GetProperty** - Safely retrieves a property value:
- Checks PSObject first using Safe-HasProperty, then falls back to hashtable `.ContainsKey()` check
- Always returns the specified default (e.g., 'Unknown' or $null) instead of throwing

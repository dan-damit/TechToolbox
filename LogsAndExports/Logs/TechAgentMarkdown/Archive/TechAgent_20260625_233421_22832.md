## about_Remove-Printers.help.txt

### NAME
Remove-Printers.ps1

### SHORT DESCRIPTION
Removes all printers from the system, with optional removal of ports, drivers, and per-user mappings.

### LONG DESCRIPTION
The Remove-Printers function uses Win32_Printer (CIM) to remove printer queues after resetting the print spooler service and clearing the spool folder. It provides options to also remove TCP/IP ports and printer drivers. 

This function includes fallbacks for provider hiccups and frees common process locks such as splwow64 and PrintIsolationHost. It can also remove per-user network printer connections across all user profiles on the system.

### PARAMETERS
- **IncludePorts**
  Also removes TCP/IP printer ports (non-standard). Use with caution as this may affect other services relying on these ports.

- **IncludeDrivers**
  Also removes printer drivers after the queues are gone. This helps in cleaning up unused driver packages from the system.

- **Force**
  Enables best-effort forced cleanup of driver packages via pnputil if standard removal fails. This is useful for stubborn drivers that do not remove easily.

- **AllUsers**
  Attempts to remove per-user network printer connections for all user profiles on the system. This ensures a comprehensive cleanup across multiple users.

- **PassThru**
  Outputs a summary object containing counts of removed printers, ports, drivers, and any failures encountered during the process. This is helpful for auditing purposes.

### EXAMPLES
#### Example 1: Comprehensive Removal
```powershell
Remove-Printers -IncludePorts -IncludeDrivers -Force -AllUsers -PassThru
```
This command removes all printers, TCP/IP ports, and drivers with forced cleanup, clears per-user connections for all users, and outputs a summary.

#### Example 2: Dry Run
```powershell
Remove-Printers -WhatIf
```
This command simulates the removal of printers without actually making any changes, allowing you to preview what would be removed.

### NOTES
- This function has a high confirm impact due to its destructive nature. Always review the output carefully.
- Ensure that no critical printing tasks are in progress before running this function.
- Administrative privileges may be required for some operations, especially when removing drivers or accessing other user profiles.

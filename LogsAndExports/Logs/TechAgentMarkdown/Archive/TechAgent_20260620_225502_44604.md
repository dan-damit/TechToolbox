## about_Invoke-RestartService

### SHORT DESCRIPTION
Restarts a named Windows service locally or on one or more remote computers.

### LONG DESCRIPTION
Invoke-RestartService performs a controlled restart of a specific service using the shared Restart-ServiceWorker logic. It initializes the TechToolbox runtime and iterates through each target specified in ComputerName.

For local targets (the current computer name or 'localhost'), it uses a local path to execute the restart. For non-local targets, it establishes a remote PowerShell session via Start-NewPSRemoteSession and Invoke-Command.

The function calls Restart-ServiceWorker with ServiceName, TimeoutSeconds, and Force parameters for each target. It cleans up any remote sessions in finally blocks and logs per-target progress and errors via Write-Log.

Invoke-RestartService supports ShouldProcess (-WhatIf/-Confirm). When WhatIf or Confirm blocks an action, the restart is skipped for that specific target.

### PARAMETERS

#### -ServiceName
Specifies the name of the Windows service to restart. This parameter is mandatory and is passed directly to Restart-ServiceWorker.

*   **Required?** True
*   **Position?** Named
*   **Default Value:** None

#### -ComputerName
Specifies one or more computer names to target. Defaults to the current computer ($env:COMPUTERNAME). Accepts values from pipeline input.

Local targets are identified as:
- Current computer name ($env:COMPUTERNAME)
- localhost

Non-local targets are handled through Start-NewPSRemoteSession and Invoke-Command.

*   **Required?** False
*   **Position?** Named
*   **Default Value:** $env:COMPUTERNAME

#### -TimeoutSeconds
Specifies the maximum number of seconds to wait for restart operations in the worker.

*   **Required?** False
*   **Position?** Named
*   **Default Value:** 30

#### -Force
Indicates that the service should be restarted forcefully. This is a switch parameter.

*   **Required?** False
*   **Position?** Named
*   **Default Value:** False

#### -Credential
Specifies a user account that has permission to perform this action. The default is the current user.

*   **Required?** False
*   **Position?** Named
*   **Default Value:** Current user credential

### INPUTS
System.String
You can pipe computer names to Invoke-RestartService via the ComputerName parameter.

### OUTPUTS
None. This function does not return any output.

### NOTES
- The function relies on Restart-ServiceWorker for the actual restart logic.
- Remote execution requires appropriate permissions and PowerShell remoting configured on target machines.
- Use -WhatIf to preview actions without executing them.

### EXAMPLES
#### Example 1: Restart a local service
```powershell
Invoke-RestartService -ServiceName 'Spooler'
```
This command restarts the Print Spooler service on the local computer.

#### Example 2: Restart a remote service with timeout
```powershell
Invoke-RestartService -ServiceName 'WSearch' -ComputerName 'Server01' -TimeoutSeconds 60
```
This command restarts the Windows Search service on Server01, waiting up to 60 seconds for completion.

#### Example 3: Force restart a service
```powershell
Invoke-RestartService -ServiceName 'Spooler' -Force
```
This command forcefully restarts the Print Spooler service.

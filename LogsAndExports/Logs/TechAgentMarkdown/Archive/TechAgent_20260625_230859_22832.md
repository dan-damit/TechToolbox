## about_Test-TTPathRoots

TOPIC
    about_Test-TTPathRoots

SHORT DESCRIPTION
    Shows effective TechToolbox path roots and key resolved config paths.

LONG DESCRIPTION
    Test-TTPathRoots returns a merged view of the active module, home, log,
    and export root directories derived from environment variables and runtime
    defaults. It is useful for verifying that TechToolbox has correctly identified
    its configuration hierarchy on the current system.

    The function determines the OneDrive root by checking $env:OneDriveCommercial,
    $env:OneDrive, $env:OneDriveConsumer, and finally falling back to $env:USERPROFILE.
    It then calculates the home root based on $env:TT_Home or defaults to a 'TechToolbox'
    subdirectory within the OneDrive root. The module root is taken from $env:TT_ModuleRoot
    or resolved via Get-ModuleRoot. Log and export roots are derived similarly, defaulting
    to 'LogsAndExports\Logs' under the home root if not explicitly set.

SYNTAX
    Test-TTPathRoots [[-EnsureDirectories]]

PARAMETERS
    -EnsureDirectories
        Creates the effective logs and exports root directories if they do not exist.

        Type: SwitchParameter
        Required: False
        Default value: $false

EXAMPLES
    Example 1:
        Test-TTPathRoots

        Description
        -----------
        Displays the current TechToolbox path roots without creating any directories.

    Example 2:
        Test-TTPathRoots -EnsureDirectories

        Description
        -----------
        Displays the current TechToolbox path roots and ensures that the logs and exports
        root directories exist, creating them if necessary.

NOTES
    - Relies on internal helper functions such as Initialize-TechToolboxRuntime,
      Get-ModuleRoot, and Join-Path.
    - The output is a PSCustomObject containing the resolved paths.

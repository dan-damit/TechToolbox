## about_Invoke-RestartService

### SHORT DESCRIPTION
Restarts a named Windows service locally or on one or more remote computers.

### LONG DESCRIPTION
Invoke-RestartService performs a controlled restart of a specific service using the shared Restart-ServiceWorker logic. It supports both local and remote execution, handling session management automatically for non-local targets.

#### Workflow Summary
- Initializes TechToolbox runtime.
- Iterates through each target in ComputerName.
- Uses a local path for the current host/localhost.
- Uses a remote PowerShell session path for non-local targets via Start-NewPSRemoteSession and Invoke-Command.
- Calls Restart-ServiceWorker with ServiceName, TimeoutSeconds, and Force parameters.
- Cleans up remote sessions in finally blocks to ensure resources are released.
- Logs per-target progress and errors via Write-Log.

#### ShouldProcess Support
The function supports -WhatIf and -Confirm. When WhatIf or Confirm blocks an action, the restart is skipped for that specific target, but processing continues for other targets if applicable.

### PARAMETERS

#### -ServiceName
Specifies the name of the Windows service to restart. This parameter is mandatory and is passed directly to Restart-ServiceWorker.

    Required?                    true
    Position?                    named
    Default value                None
    Accept pipeline input?       false
    Accept wildcard characters?  false

#### -ComputerName
Specifies one or more computer names to target. Defaults to the current computer ($env:COMPUTERNAME). This parameter accepts values from pipeline input.

Local targets are identified as:
- Current computer name ($env:COMPUTERNAME)

### EXAMPLES

#### Example 1: Restart a local service

This command restarts the Print Spooler service on the local computer.

#### Example 2: Restart a remote service with timeout

This command restarts the Windows Update service on Server01 and Server02, waiting up to 60 seconds for each operation.

#### Example 3: Use pipeline input

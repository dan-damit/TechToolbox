## about_Get-TTPathRoots

SHORT DESCRIPTION
    Shows effective TechToolbox path roots and key resolved config paths.

LONG DESCRIPTION
    The Test-TTPathRoots function (referenced here as Get-TTPathRoots for help consistency) returns the active module, home, log, and export root directories derived from environment variables and runtime defaults. It helps users understand where TechToolbox stores its data and configuration.

    Path Resolution Logic:
    - Home Root: Uses $env:TT_Home if set; otherwise falls back to OneDriveCommercial, OneDrive, OneDriveConsumer, or USERPROFILE joined with 'TechToolbox'.
    - Module Root: Uses $env:TT_ModuleRoot if set; otherwise calls Get-ModuleRoot.
    - Logs Root: Uses $env:TT_LogsRoot if set; otherwise joins Home Root with 'LogsAndExports\Logs'.

PARAMETER EnsureDirectories
    Creates the effective logs and exports root directories if they do not exist. This is a switch parameter.

INPUTS
    None. You cannot pipe objects to Test-TTPathRoots.

OUTPUTS
    System.Management.Automation.PSCustomObject. The function returns an object containing the resolved path roots.

EXAMPLE 1: Show Path Roots
    Test-TTPathRoots

    Description:
    Displays the current effective TechToolbox path roots without creating any directories.

EXAMPLE 2: Ensure Directories Exist
    Test-TTPathRoots -EnsureDirectories

    Description:
    Displays the paths and creates the logs and exports root directories if they are missing.

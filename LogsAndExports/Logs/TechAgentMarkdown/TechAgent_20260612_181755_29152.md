## Using the PowerShell Help System to Display File Content

To display the content of an `about_*.help.txt` file using the native PowerShell help system, use the `Get-Help` cmdlet.

-> Basic Command Syntax
```powershell
Get-Help about_<topic_name>
```

Example (for the `Invoke-TechAgent` topic):
```powershell
Get-Help about_Invoke_TechAgent
```

## Important Notes for Console Use

- Hyphens vs Underscores: PowerShell normalizes hyphens to underscores when resolving topic names. You type `about_Invoke_TechAgent` even if the conceptual name uses a hyphen.
- Discoverability: The help system only scans specific directories by default (typically `$pshome\en-US` and standard module paths). If your custom `.help.txt` file was saved to a different location, `Get-Help` will return "No help found".
- Making Custom Files Visible: To register your file with the help system:
  - Move the file into a module's directory structure: `<ModulePath>\<ModuleName>\en-US\about_<Topic>.help.txt`
  - Run `Update-Help` in your console to cache the new topic.
  - Alternatively, add the folder to `$env:PSModulePath` before importing the help content.

-> Alternative for Raw Text Viewing
If you do not need the file registered in the help system and just want to dump the raw text into the console, use `Get-Content`:
```powershell
Get-Content "C:\Path\To\en-US\about_Invoke_TechAgent.help.txt"
```

## Overview

`Invoke-TechAgent` is a function that calls the TechToolbox.Agent C# runtime and prints the agent's response. It allows users to interact with a local AI agent using natural language instructions, supporting various models and iteration limits for complex tasks.

> **Note:** This function requires the TechToolbox Agent runtime to be installed and configured correctly.

---

## Parameters

| Parameter | Type | Description |
|----------|------|-------------|
| **Prompt** | `string` | **[Mandatory]** The natural-language instruction for the agent. |
| **Model** | `string` | Optional Ollama model name (for example: llama3, mistral, qwen2.5-coder). |
| **MaxIterations** | `int` | Maximum number of tool/reasoning iterations before the agent concludes. Valid range: 1-500. Defaults to `15`. |
| **Quiet** | `switch` | Legacy compatibility switch. Agent traces are now suppressed by default. |
| **ConfirmDestructive** | `switch` | Explicitly authorizes destructive operations for this run. |
| **SignedFilePolicy** | `string` | Policy to use when overwriting an existing Authenticode-signed PowerShell file. 'ignore' blocks the overwrite and 'strip' allows the overwrite while removing the signature block text. |
| **AutoRetryOnRecursion** | `switch` | Enables a single automatic retry when the C# agent hits an iteration limit. |
| **DisableAutoRetryOnRecursion** | `switch` | Disables recursion-limit auto-retry for this invocation, overriding environment defaults. |
| **NoTranscript** | `switch` | Disables the per-run console transcript log. |

---

## How It Works

### **Execution Flow**
1. The function initializes the TechToolbox.Agent C# runtime.
2. It sends the provided `-Prompt` to the specified AI model (if provided) or the default configured model.
3. The agent processes the prompt, potentially using tools in a loop up to `-MaxIterations` times.
4. If recursion limits are hit and `AutoRetryOnRecursion` is enabled, it attempts one automatic retry.
5. The final response from the agent is printed to the console.

### **Destructive Operations**
To perform destructive actions (e.g., deleting files, modifying system settings), you must explicitly pass the `-ConfirmDestructive` switch. Without this switch, the agent will refuse to execute dangerous commands.

---

## Examples

### **Basic Usage**
Ask the agent a simple question or give it a task.
```powershell
Invoke-TechAgent -Prompt "What is the current date and time?"
```

### **Using a Specific Model**
Specify an Ollama model for the request.
```powershell
Invoke-TechAgent -Prompt "Explain how PowerShell pipelines work" -Model "qwen2.5-coder"
```

### **Allowing Destructive Actions**
Authorize the agent to perform destructive operations (use with caution).
```powershell
Invoke-TechAgent -Prompt "Delete all files in C:\Temp\OldLogs" -ConfirmDestructive
```

### **Increasing Iteration Limit**
For complex tasks requiring more reasoning steps, increase the max iterations.
```powershell
Invoke-TechAgent -Prompt "Analyze this script for security vulnerabilities and suggest fixes." -MaxIterations 50
```

---

## Notes

- The `-Quiet` switch is provided for legacy compatibility; traces are suppressed by default in newer versions.
- If the agent hits an iteration limit, it may conclude prematurely. Use `-MaxIterations` to allow more steps if needed.
- Ensure the target Ollama model specified in `-Model` is installed and running locally.
- The `SignedFilePolicy` parameter only applies when the function itself overwrites a signed script file; it does not affect the agent's internal operations.

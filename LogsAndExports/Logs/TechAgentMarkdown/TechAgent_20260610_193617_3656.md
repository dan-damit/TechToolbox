## 2. Medium-Impact Improvements

### D. Robustify Ollama Model Parsing

**Current behavior:** The `ollama list` output is parsed by splitting on whitespace after a `NAME` header line. This is fragile if model names contain spaces, tabs, or unusual formatting changes in future Ollama versions.

**Why it matters:** Could cause false "model not available" errors even when the model exists.

**Recommendation:** Use a more robust parser:
```powershell
$availableModels = @()
foreach ($line in $ollamaListOutput) {
    if ($line -match '^\S+\s+') {
        $parts = $line -split '\s+', 2
        if (-not [string]::IsNullOrWhiteSpace($parts[0])) {
            $availableModels += $parts[0]
        }
    }
}
```
Alternatively, consider using `ollama list --json` (if supported in the target Ollama version) to get structured output that is immune to format changes.

### E. Simplify Configuration Property Checks

**Current behavior:** The markdown enabled check uses `$cfg.transcript.PSObject.Properties['markdownEnabled']` and similar verbose patterns.

**Why it matters:** Hard to read, hard to maintain.

**Recommendation:** Use direct property access with null-coalescing:
```powershell
$markdownEnabled = if ($null -ne $cfg.transcript.markdownEnabled) { [bool]$cfg.transcript.markdownEnabled } else { $true }
```

This is cleaner and follows PowerShell conventions better.

### F. Stream Output During Execution for Better UX

**Current behavior:** stdout/stderr are captured only after the process exits via `$stdoutTask.GetAwaiter().GetResult()`. Users see no output until completion.

**Why it matters:** For long-running agent sessions (30+ minutes), users have no feedback that work is progressing. Also, debugging hangs becomes harder.

**Recommendation:** Consider a hybrid approach:
- Stream stdout in chunks to the transcript console using a non-blocking read loop with a small timeout.
- Keep full capture for the final log write.
This requires careful deadlock prevention but significantly improves user experience.
